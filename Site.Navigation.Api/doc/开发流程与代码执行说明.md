# Site.Navigation.Api 开发流程与代码执行说明

本文说明：服务如何启动、请求如何流转、各目录职责，以及新增一个业务模块的标准步骤。

---

## 一、启动 `main.go` 做了哪些事

执行 `go run main.go`（或编译后运行）时，顺序如下：

```
main()
  │
  ├─ 1. config.InitConfig()
  │     读取 config/config.json（数据库、JWT、Redis、端口）
  │
  ├─ 2. utils.InitJWT(...)
  │     用配置里的 jwt.secret / expire_hours 初始化 Token 签发与校验
  │
  ├─ 3. config.InitDB()
  │     用 database.dsn 直连 MySQL（GORM）
  │     defer CloseDB()：进程退出时关连接
  │
  ├─ 4. 若 redis.enabled = true → config.InitRedis()
  │     连接 Redis；否则跳过
  │     defer CloseRedis()
  │
  ├─ 5. gin.Default()
  │     创建 HTTP 引擎（自带 Logger、Recovery）
  │
  ├─ 6. router.RegisterRoutes(r)
  │     注册 /health、/api/v1/... 全部路由
  │
  └─ 7. r.Run(port)
        监听端口（默认 :18080），开始接收请求
```

**要点：**

- 配置只来自本地 `config/config.json`，不连 Consul。
- 数据库、Redis 在启动阶段就连上；连不上会直接 `Fatal` 退出。
- 路由注册完成后才真正对外提供接口。

## 二、一次 HTTP 请求的执行流程

以 `POST /api/v1/prompts/UpdateItem`（需登录）为例：

```
客户端请求
    │
    ▼
Gin Engine
    │
    ▼
路由匹配 → /api/v1 路由组
    │
    ▼
middleware.AuthMiddleware()          ← 全局鉴权
    │  ├─ 若在匿名白名单 → 放行
    │  └─ 否则校验 Authorization: Bearer <token>
    │       通过后写入上下文：user_id、account
    ▼
controller.UpdatePromptItem          ← 控制器：绑参、调服务、写 HTTP 响应
    │
    ▼
service.PromptService.UpdateItem     ← 服务：业务规则、调数据库
    │
    ▼
GORM → MySQL（t_prompt_item）
    │
    ▼
JSON 响应返回客户端
```

匿名接口（如 `GET /api/v1/prompts/GetTree`）同样经过 `AuthMiddleware`，但因注册时 `AllowAnonymous: true`，会跳过 Token 校验。

健康检查 `GET /health` 不在 `/api/v1` 组内，**不走 JWT 中间件**。

## 三、目录结构说明

```
Site.Navigation.Api/
├── main.go                 # 程序入口：配置 → DB/Redis → 路由 → 监听
├── go.mod                  # Go 模块与依赖
├── config/                 # 配置与基础设施连接
├── model/                  # 数据结构（表模型 + 请求/响应 DTO）
├── service/                # 业务逻辑（不直接写 HTTP）
├── controller/             # HTTP 入口（绑参、调 service、返回 JSON）
├── router/                 # 路由注册、AllowAnonymous 辅助
├── middleware/             # 中间件（JWT、匿名白名单）
├── utils/                  # 通用工具（JWT、Redis Helper）
├── scripts/                # MySQL 建表 SQL
├── doc/                    # 项目文档（本文件所在目录）
└── README.md               # 快速开始说明
```

### 各文件夹职责

| 目录 | 做什么 | 不该做什么 |
|------|--------|------------|
| **config** | 读 `config.json`；初始化/关闭 MySQL、Redis | 写业务逻辑、拼业务 SQL |
| **model** | 定义表结构、`TableName()`、请求体/响应体 | 写复杂业务、操作 HTTP |
| **service** | 业务规则、事务、调 GORM/Redis | 直接 `c.JSON`、解析 Gin Context |
| **controller** | `ShouldBindJSON`、调 service、按错误码返回 | 堆业务细节、直接拼复杂 SQL |
| **router** | 把 URL 绑到 controller；标记是否匿名 | 写业务代码 |
| **middleware** | 鉴权、跨切面逻辑 | 业务 CRUD |
| **utils** | 可复用工具（JWT、Redis 封装） | 依赖具体业务表 |
| **scripts** | 建表/初始化数据 SQL | — |
| **doc** | 开发说明文档 | — |

### 分层调用约定

```
router  →  middleware  →  controller  →  service  →  model / DB / Redis
```

**只允许自上而下调用**，避免 `service` 反过来依赖 `controller`。

## 四、日常开发流程（建议）

1. **改配置**（如有需要）  
   编辑 `config/config.json`：`database.dsn`、`redis`、`jwt`、`server.port`。

2. **改数据库**（如有新表/字段）  
   在 `scripts/` 增加 SQL，在 MySQL 中执行。

3. **写代码**（见下一节「新模块」）  
   `model` → `service` → `controller` → `router`。

4. **启动调试**

   ```bash
   cd Site.Navigation.Api
   go mod tidy
   go run main.go
   ```

5. **调接口**  
   - 匿名：直接请求  
   - 需登录：先 `POST /api/v1/auth/login` 拿 `token`，再带  
     `Authorization: Bearer <token>`

## 五、如何开发一个新模块（标准步骤）

下面以新增「书签 Bookmark」为例，按现有风格照做即可。

### 步骤 1：建表（scripts）

新增例如 `scripts/03_t_bookmark.sql`：

```sql
CREATE TABLE IF NOT EXISTS `t_bookmark` (
  `id` INT NOT NULL AUTO_INCREMENT,
  `title` VARCHAR(100) NOT NULL,
  `url` VARCHAR(500) NOT NULL,
  `is_deleted` TINYINT NOT NULL DEFAULT 0,
  `create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

在目标库执行该 SQL。

### 步骤 2：模型（model）

新建 `model/bookmark.go`：

```go
package model

import "time"

type Bookmark struct {
	ID         int       `json:"id" gorm:"column:id;primaryKey;autoIncrement"`
	Title      string    `json:"title" gorm:"column:title;size:100;not null"`
	URL        string    `json:"url" gorm:"column:url;size:500;not null"`
	IsDeleted  bool      `json:"is_deleted" gorm:"column:is_deleted;default:0;not null"`
	CreateTime time.Time `json:"create_time" gorm:"column:create_time"`
}

func (Bookmark) TableName() string { return "t_bookmark" }

type CreateBookmarkRequest struct {
	Title string `json:"title" binding:"required"`
	URL   string `json:"url" binding:"required"`
}
```

### 步骤 3：服务（service）

新建 `service/bookmark_service.go`：

```go
package service

import (
	"sitenavigation/model"
	"gorm.io/gorm"
)

type BookmarkService struct{ db *gorm.DB }

func NewBookmarkService(db *gorm.DB) *BookmarkService {
	return &BookmarkService{db: db}
}

func (s *BookmarkService) Create(req *model.CreateBookmarkRequest) (*model.Bookmark, error) {
	item := &model.Bookmark{Title: req.Title, URL: req.URL}
	if err := s.db.Create(item).Error; err != nil {
		return nil, err
	}
	return item, nil
}

func (s *BookmarkService) List() ([]*model.Bookmark, error) {
	var list []*model.Bookmark
	err := s.db.Where("is_deleted = ?", false).Order("id desc").Find(&list).Error
	return list, err
}
```

### 步骤 4：控制器（controller）

新建 `controller/bookmark_controller.go`：

```go
package controller

import (
	"net/http"
	"sitenavigation/config"
	"sitenavigation/model"
	"sitenavigation/service"
	"github.com/gin-gonic/gin"
)

var bookmarkService *service.BookmarkService

func getBookmarkService() *service.BookmarkService {
	if bookmarkService == nil {
		bookmarkService = service.NewBookmarkService(config.DB)
	}
	return bookmarkService
}

func CreateBookmark(c *gin.Context) {
	var req model.CreateBookmarkRequest
	if err := c.ShouldBindJSON(&req); err != nil {
		c.JSON(http.StatusBadRequest, gin.H{"error": err.Error()})
		return
	}
	item, err := getBookmarkService().Create(&req)
	if err != nil {
		c.JSON(http.StatusInternalServerError, gin.H{"error": err.Error()})
		return
	}
	c.JSON(http.StatusOK, item)
}

func GetBookmarkList(c *gin.Context) {
	list, err := getBookmarkService().List()
	if err != nil {
		c.JSON(http.StatusInternalServerError, gin.H{"error": err.Error()})
		return
	}
	c.JSON(http.StatusOK, list)
}
```

### 步骤 5：注册路由（router）

在 `router/router.go` 的 `v1` 组内增加：

```go
bookmarks := v1.Group("/bookmarks")
{
	// 需要登录
	bookmarks.POST("/Create", controller.CreateBookmark)
	bookmarks.GET("/GetList", controller.GetBookmarkList)

	// 若允许匿名，用 RegisterRoute：
	// RegisterRoute(bookmarks, RouteConfig{
	//   Method: "GET", Path: "/GetList",
	//   Handler: controller.GetBookmarkList,
	//   AllowAnonymous: true,
	// })
}
```

## 六、现有模块对照（抄作业用）

| 模块 | model | service | controller | 路由前缀 |
|------|--------|---------|------------|----------|
| 用户/登录 | `user.go` | `user_service.go` | `auth_controller.go` / `user_controller.go` | `/auth` `/users` |
| AI 模板 | `prompt.go` | `prompt_service.go` | `prompt_controller.go` | `/prompts` |
| 健康检查 | — | — | `health_controller.go` | `/health` |

开发新模块时，优先对照 **AI 模板（prompt）** 这一套：表结构清晰，含列表树、增删改。

## 七、鉴权约定（写路由时必看）

- `/api/v1` 下默认都要 JWT。
- 匿名接口用 `RegisterRoute(..., AllowAnonymous: true)`，会写入白名单。
- Token 来自登录接口：`POST /api/v1/auth/login`  
  请求头格式：`Authorization: Bearer <token>`。

## 九、推荐阅读顺序（新人）

1. `main.go` — 启动顺序  
2. `config/config.json` + `config/config.go` — 配置从哪来  
3. `router/router.go` — 有哪些接口  
4. `middleware/auth.go` — 鉴权怎么拦  
5. `controller/prompt_controller.go` → `service/prompt_service.go` → `model/prompt.go` — 完整业务样例  

按上面路径走一遍，再按第五节加新模块，即可保持项目风格一致。
